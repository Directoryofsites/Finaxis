📑 Contenido

    🎯 Introducción

    El módulo de Registro de Pagos permite asentar todos los recaudos recibidos de los Deudores, aplicándolos automáticamente a las cuentas pendientes. El sistema maneja tanto pagos individuales por Préstamo como pagos consolidados por Deudor.

    💡 Objetivo: Registrar de forma precisa todos los pagos recibidos, aplicándolos correctamente a las obligaciones pendientes y manteniendo actualizada la cartera.

    Tipos de Pagos

    • 💳 Pago Individual: Pago específico de una Préstamo
    • 🏢 Pago Consolidado: Pago de múltiples Préstamos de un Deudor
    • 💰 Pago Parcial: Abono a cuenta sin cubrir el total
    • ✅ Pago Total: Cubre completamente las obligaciones
    • 📅 Pago Anticipado: Pago antes del vencimiento

    Formas de Pago Soportadas

    • 💵 Efectivo
    • 🏦 Transferencia bancaria
    • 💳 Tarjeta de crédito/débito
    • 📄 Cheque
    • 📱 Pago electrónico (PSE, Nequi, etc.)

    🚪 Cómo Acceder

    1. Ingrese al sistema ContaPY2
    2. Navegue al módulo "Gestión de Recaudos"
    3. Haga clic en "Registro de Pagos" o "Recaudos (Pagos)"

    💳 Pago Individual por Préstamo

    Proceso de Registro

    1. Buscar Préstamo: Localice la Préstamo que está pagando
    2. Consultar Cartera: Revise las obligaciones pendientes
    3. Ingresar Pago: Registre el monto y forma de pago
    4. Aplicar Pago: El sistema aplica automáticamente
    5. Confirmar: Verifique y confirme el registro

    Información Requerida

    Campo Descripción Requerido Ejemplo
    Préstamo Préstamo que realiza el pago ✅ Sí APT-101
    Fecha de Pago Cuándo se recibió el pago ✅ Sí 2024-03-15
    Valor Pagado Monto recibido ✅ Sí $250,000
    Forma de Pago Método utilizado ✅ Sí Transferencia
    Referencia Número de transacción ❌ No TRF-123456
    Observaciones Notas adicionales ❌ No Pago mes de marzo

    Aplicación Automática

    El sistema aplica los pagos siguiendo este orden de prioridad:

    1. Intereses de Mora: Primero se cubren los intereses
    2. Saldos Más Antiguos: Se paga desde lo más vencido
    3. Por Concepto: Según configuración de prioridades
    4. Saldo Restante: Se deja como saldo a favor si sobra
    💡 Consejo: Si el pago no cubre todo el saldo, se aplicará proporcionalmente a todas las obligaciones pendientes.

    🏢 Pago Consolidado por Deudor

    ¿Cuándo Usar Pago Consolidado?

    Use esta opción cuando:

    • 🏠 Un Deudor tiene múltiples Préstamos
    • 💰 Realiza un solo pago para todas sus Préstamos
    • 🔄 Quiere distribuir el pago entre sus obligaciones
    • ⚖️ Necesita compensar saldos a favor con deudas

    Proceso de Pago Consolidado

    1. Buscar Deudor: Localice el Deudor en el sistema
    2. Ver Resumen: Consulte todas sus Préstamos y saldos
    3. Registrar Pago: Ingrese el monto total recibido
    4. Distribuir: El sistema distribuye automáticamente
    5. Ajustar: Modifique la distribución si es necesario
    6. Confirmar: Aplique el pago a todas las Préstamos

    Distribución Automática

    El sistema distribuye el pago siguiendo estas reglas:

    Prioridad Concepto Criterio
    1 Intereses de Mora Todos los intereses primero
    2 Saldos Vencidos Desde el más antiguo
    3 Saldos Corrientes Obligaciones del período actual
    4 Saldo a Favor Excedente queda como anticipo

    Distribución Manual

    También puede distribuir manualmente:

    1. Seleccione "Distribución Manual"
    2. Asigne montos específicos a cada Préstamo
    3. Verifique que la suma coincida con el pago total
    4. Confirme la distribución personalizada
    ⚠️ Importante: En distribución manual, asegúrese de que la suma de los montos asignados sea igual al valor total del pago.

    ⚠️ Errores Comunes y Soluciones

    Error: "Préstamo no encontrada"

    Problema: No se puede localizar la Préstamo para registrar el pago.
    Causa: Código de Préstamo incorrecto o Préstamo inactiva.
    Solución:
    • Verifique el código de la Préstamo
    • Use el buscador para localizar la Préstamo
    • Confirme que la Préstamo esté activa
    • Revise si hay errores de digitación

    Error: "Valor de pago inválido"

    Problema: El sistema no acepta el valor del pago.
    Causa: Formato incorrecto o valor negativo.
    Solución:
    • Use solo números, sin puntos ni comas
    • Asegúrese de que sea un valor positivo
    • No use símbolos de moneda ($)
    • Verifique que no exceda límites del sistema

    Error: "Fecha de pago inválida"

    Problema: No se puede registrar el pago con esa fecha.
    Causa: Fecha futura o muy antigua.
    Solución:
    • Use fechas dentro del rango permitido
    • No use fechas futuras
    • Verifique el formato de fecha (DD/MM/YYYY)
    • Consulte con administrador sobre períodos cerrados

    Problema: "El pago no se aplicó correctamente"

    Problema: El pago se registró pero no se aplicó a las cuentas.
    Causa: Error en la configuración contable o proceso interrumpido.
    Solución:
    • Verifique el estado del pago en el historial
    • Consulte si se generaron los movimientos contables
    • Revise la configuración de cuentas en los conceptos
    • Contacte soporte si el problema persiste

    Problema: "Diferencias en la distribución consolidada"

    Problema: La suma de la distribución no coincide con el pago total.
    Causa: Error en la distribución manual o cálculo automático.
    Solución:
    • Verifique que la suma de distribución sea igual al pago total
    • Revise si hay redondeos que causen diferencias
    • Use la distribución automática si hay dudas
    • Ajuste manualmente las diferencias menores

    Problema: "No se puede anular el pago"

    Problema: Necesita anular un pago registrado incorrectamente.
    Causa: Restricciones del sistema o permisos insuficientes.
    Solución:
    • Verifique que tenga permisos de anulación
    • Confirme que el pago no esté en un período cerrado
    • Use la función de "Reversar Pago" si está disponible
    • Contacte al administrador para anulaciones especiales
