🎯 Introducción

El módulo de Gestión de Préstamos permite administrar todos los activos sujetos a cobro en su Cartera de Crédito. Esto incluye apartamentos, casas, locales comerciales, parqueaderos, depósitos y cualquier otro tipo de Préstamo que genere obligaciones de pago.

💡 Objetivo: Mantener un registro completo y actualizado de todas las Préstamos, sus características, Deudores y configuraciones de cobro.

¿Qué puede hacer en este módulo?

  • ✅ Crear y registrar nuevas Préstamos
  • ✅ Editar información de Préstamos existentes
  • ✅ Asignar Deudores a las Préstamos
  • ✅ Configurar coeficientes de participación
  • ✅ Gestionar módulos de cobro
  • ✅ Realizar acciones masivas en múltiples Préstamos
  • ✅ Exportar listados a PDF
  • ✅ Filtrar y buscar Préstamos
🌟 Novedad: Metamorfosis Multi-Sectorial

El sistema ahora es inteligente y se adapta a su sector. Si su entidad es un Colegio, los botones y títulos hablarán de "Estudiantes" y "Grados". Si es una Cooperativa de Transporte, hablarán de "Vehículos" y "Rutas". Todo de forma automática.

🚪 Cómo Acceder

  1. Ingrese al sistema ContaPY2
  2. Navegue al módulo "Gestión de Recaudos"
  3. Haga clic en "Préstamos" o "Préstamos / Activos"
📋 Nota: Dependiendo de la configuración de su sistema, esta opción puede aparecer como "Préstamos", "Préstamos / Activos" o simplemente "Préstamos".

🏗️ Módulos de Cobro

Los módulos permiten aplicar diferentes conceptos de cobro según el tipo de Préstamo:

  • Residencial: Para apartamentos y casas
  • Comercial: Para locales comerciales
  • Parqueadero: Para espacios de estacionamiento
  • Depósito: Para bodegas y cuartos útiles
  1. Seleccione los módulos que aplican para esta Préstamo
  2. Los módulos determinan qué conceptos de cobro se aplicarán
  3. Puede modificar los módulos posteriormente

Validaciones del Sistema

⚠️ Validaciones Importantes:
  • El código de Préstamo debe ser único en el sistema
  • El coeficiente debe ser un número decimal positivo
  • La suma de todos los coeficientes debería ser 100%
  • Todos los campos marcados como requeridos deben completarse

✏️ Editar Préstamo Existente

Proceso de Edición

  1. En el listado de Préstamos, localice la Préstamo que desea editar
  2. Haga clic en el icono de editar (✏️) en la columna de acciones
  3. Modifique los campos necesarios en el formulario
  4. Guarde los cambios

Campos Editables

Puede modificar todos los campos de la Préstamo excepto:

  • ❌ ID interno del sistema (se genera automáticamente)
  • ✅ Código (con precaución, puede afectar reportes históricos)
  • ✅ Tipo de Préstamo
  • ✅ Coeficiente de participación
  • ✅ Deudor asignado
  • ✅ Línea o grupo
  • ✅ Aplica Beneficio de Descuento por Pronto Pago (ON/OFF)
  • ✅ Módulos de cobro
⚠️ Precaución al Editar:
  • Cambiar el código puede afectar reportes históricos
  • Modificar el coeficiente afecta futuros cobros
  • Cambiar módulos puede alterar los conceptos aplicables

Cambio de Deudor

  1. En el formulario de edición, haga clic en "Cambiar Deudor"
  2. Busque y seleccione el nuevo Deudor
  3. Confirme el cambio
  4. El sistema registrará el historial del cambio
💡 Historial: El sistema mantiene un registro de todos los cambios de Deudor para efectos de auditoría y reportes.

🗑️ Eliminar Préstamos

Proceso de Eliminación

  1. Localice la Préstamo que desea eliminar en el listado
  2. Haga clic en el icono de eliminar (🗑️) en la columna de acciones
  3. Confirme la eliminación en el diálogo que aparece
  4. La Préstamo será removida del sistema
⚠️ IMPORTANTE - Restricciones de Eliminación:

No podrá eliminar una Préstamo si:

  • Tiene movimientos contables asociados
  • Tiene facturas generadas
  • Tiene pagos registrados
  • Tiene saldo pendiente

Solución: En estos casos, considere marcar la Préstamo como "Inactiva" en lugar de eliminarla.

Alternativa: Desactivar Préstamo

Si no puede eliminar la Préstamo por restricciones del sistema:

  1. Edite la Préstamo
  2. Cambie el estado a "Inactiva" (si está disponible)
  3. O agregue un prefijo como "INACTIVA-" al código
  4. Esto mantendrá el historial pero la excluirá de procesos futuros

⚡ Acciones Masivas

¿Qué son las Acciones Masivas?

Las acciones masivas permiten modificar múltiples Préstamos simultáneamente, ahorrando tiempo cuando necesita hacer cambios similares en varias Préstamos.

Proceso de Acciones Masivas

  1. Seleccionar Préstamos: Use los checkboxes para marcar las Préstamos que desea modificar
  2. Activar Modo Masivo: Aparecerá un botón "Acciones Masivas" cuando tenga Préstamos seleccionadas
  3. Configurar Cambios: En el modal que aparece, configure qué módulos agregar o remover
  4. Aplicar Cambios: Confirme la operación para aplicar los cambios

Tipos de Acciones Masivas Disponibles

🏗️ Gestión de Módulos

La acción masiva más común es la gestión de módulos de cobro:

Acción Descripción Ejemplo de Uso
🟢 Asignar Módulo Agregar un módulo a las Préstamos seleccionadas Agregar módulo "Parqueadero" a apartamentos
🔴 Remover Módulo Quitar un módulo de las Préstamos seleccionadas Remover módulo "Comercial" de apartamentos
🔄 Combinado Asignar y remover módulos en la misma operación Cambiar de "Residencial" a "Comercial"

Interfaz de Acciones Masivas

  1. Selección Visual: Haga clic en los módulos para cambiar su estado:
    • 🔘 Gris: Sin cambios
    • 🟢 Verde: Asignar módulo
    • 🔴 Rojo: Remover módulo
  2. Confirmación: Revise el resumen antes de aplicar
  3. Aplicación: Los cambios se aplican inmediatamente
✅ Ejemplo Práctico: Si necesita agregar el módulo "Parqueadero" a 50 apartamentos, seleccione los 50 apartamentos, active acciones masivas, marque "Parqueadero" en verde y aplique. ¡Listo en segundos!

Consejos para Acciones Masivas

  • 💡 Use filtros para seleccionar Préstamos similares más fácilmente
  • 💡 Revise siempre el resumen antes de confirmar
  • 💡 Haga pruebas con pocas Préstamos primero
  • 💡 Mantenga respaldos antes de cambios masivos importantes

⚠️ Errores Comunes y Soluciones

Error: "Código de Préstamo ya existe"

Problema: Al crear una Préstamo, aparece el mensaje "El código ya existe".
Causa: Está intentando usar un código que ya está asignado a otra Préstamo.
Solución:
  • Verifique el listado de Préstamos existentes
  • Use un código diferente y único
  • Considere usar un formato estándar (ej: APT-101, LOC-001)

Error: "Coeficiente inválido"

Problema: El sistema no acepta el coeficiente ingresado.
Causa: El formato del coeficiente no es correcto.
Solución:
  • Use formato decimal: 0.8500 (no 0.85% ni 85%)
  • Use punto como separador decimal, no coma
  • Asegúrese de que sea un número positivo

Error: "No se puede eliminar la Préstamo"

Problema: El sistema impide eliminar una Préstamo.
Causa: La Préstamo tiene movimientos contables o facturas asociadas.
Solución:
  • Verifique si tiene saldos pendientes
  • Consulte el historial de movimientos
  • Considere desactivar en lugar de eliminar
  • Contacte al administrador si es necesario eliminar

Problema: "Las acciones masivas no funcionan"

Problema: No aparece el botón de acciones masivas.
Causa: No ha seleccionado ninguna Préstamo.
Solución:
  • Seleccione al menos una Préstamo usando los checkboxes
  • El botón aparecerá automáticamente
  • Verifique que tenga permisos para acciones masivas

Problema: "La búsqueda no encuentra Préstamos"

Problema: El filtro de búsqueda no muestra resultados.
Solución:
  • Verifique que esté escribiendo correctamente
  • La búsqueda requiere al menos 2 caracteres
  • Limpie los filtros y intente de nuevo
  • Actualice la página si persiste el problema
