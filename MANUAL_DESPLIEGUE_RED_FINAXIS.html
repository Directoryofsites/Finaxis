Fase A: Configuración del Servidor

🏠Paso 1: Perfil de Red del Servidor

Configuramos la red como "Privada" para habilitar el descubrimiento.

  • Presiona la tecla Windows en tu teclado.
  • Escribe la palabra PowerShell.
  • Haz Clic Derecho sobre "Windows PowerShell".
  • Selecciona Ejecutar como administrador.
  • Copia y pega este comando:
Get-NetConnectionProfile | Set-NetConnectionProfile -NetworkCategory Private

🛡️Paso 2: Apertura de Puertos (Firewall)

Autorizamos la entrada de datos al servidor.

  • En la misma ventana de PowerShell (como administrador), pega este comando:
New-NetFirewallRule -DisplayName "Finaxis Network" -Direction Inbound -LocalPort 3000,8765,5432 -Protocol TCP -Action Allow

🔍Paso 3: Identificar la IP del Servidor

  • En la ventana de PowerShell, escribe: ipconfig y presiona Enter.
  • Busca la línea que dice Dirección IPv4.
  • Anota ese número (Ejemplo: 192.168.40.133).
Esta es la IP que usarás en todos los equipos clientes.

🐘Paso 4: Configuración de PostgreSQL

  1. Abre C:\Program Files\PostgreSQL\17\data\postgresql.conf y busca listen_addresses. Cámbialo a '*'.
  2. Abre C:\Program Files\PostgreSQL\17\data\pg_hba.conf y al final agrega:
host all all [TU_IP_SIN_EL_ULTIMO_NUMERO].0/24 scram-sha-256

Ejemplo: Si tu IP es 192.168.40.133, pones: 192.168.40.0/24

REINICIA el servicio de PostgreSQL desde el administrador de tareas o Servicios de Windows.
Fase B: Configuración del Equipo Cliente

💻Paso 5: Configuración del PC Usuario

Cada equipo cliente debe confiar en la red local.

  • En el PC del Usuario, presiona Windows, busca PowerShell.
  • Haz Clic Derecho y elige Ejecutar como administrador.
  • Ejecuta:
Get-NetConnectionProfile | Set-NetConnectionProfile -NetworkCategory Private

🛡️Paso 6: Permiso de Salida (Cliente)

Aseguramos que el cliente pueda enviar datos hacia el servidor.

  • En el PowerShell del Cliente (Administrador), pega esto:
New-NetFirewallRule -DisplayName "Finaxis Client Access" -Direction Outbound -LocalPort 3000,8765,5432 -Protocol TCP -Action Allow

🚀Paso 7: Prueba y Acceso Final

  1. En el navegador del cliente prueba: http://[IP_SERVIDOR]:8765/api/ping
  2. Si ves {"status": "alive"}, entra a: http://[IP_SERVIDOR]:3000
Fase C: Seguridad y Mantenimiento

🔒Paso 8: Mantenimiento de la Seguridad

¿Mi equipo está desprotegido? No. El Firewall de Windows sigue activo y bloqueando todo lo demás. Solo hemos creado una "excepción" para Finaxis.

  • Si algún día deseas ELIMINAR el acceso de red, abre PowerShell como Administrador y ejecuta:
Remove-NetFirewallRule -DisplayName "Finaxis Network"

Esto cerrará los puertos inmediatamente en el servidor.

🌍Paso 9: Seguridad en Viajes (Modo Público)

Si sacas tu computador de la oficina y lo conectas en un aeropuerto o café internet, es vital volver al modo Público:

  • Abre PowerShell como Administrador.
  • Ejecuta:
Get-NetConnectionProfile | Set-NetConnectionProfile -NetworkCategory Public

Nota: Cuando regreses a la oficina, deberás volver a ejecutar el Paso 1 de este manual para que el sistema funcione de nuevo.

Fase D: Anexo de Emergencia (Rescate de Postgres)

🚑Problema: Error de Contraseña al Instalar

Si al instalar PostgreSQL en un equipo nuevo el sistema te rechaza la clave (Error: FATAL: authentication failed), sigue este protocolo de rescate:

1. Abrir el "Candado" (Modo Trust)

  • Busca el Bloc de Notas en Windows.
  • Haz Clic Derecho y elige Ejecutar como Administrador.
  • Abre el archivo: C:\Program Files\PostgreSQL\17\data\pg_hba.conf
  • Busca la línea que dice 127.0.0.1/32 y cambia scram-sha-256 por la palabra trust.
  • Guarda el archivo.

2. Reiniciar el Motor

  • Busca Servicios en Windows.
  • Busca postgresql-x64-17, clic derecho y Reiniciar.

3. Configurar en pgAdmin 4

  • Abre pgAdmin 4. Ahora entrará sin pedir clave.
  • Crear Base de Datos: Clic derecho en Databases > Create > Database > Nombre: finaxis_network > Save.
  • Poner Clave Real: Ve a Login/Group Roles > Clic derecho en postgres > Properties > Definition > Escribe tu clave en Password > Save.

4. Restaurar Seguridad (MUY IMPORTANTE)

  • Vuelve al archivo pg_hba.conf en el Bloc de Notas.
  • Cambia trust de nuevo por scram-sha-256.
  • Guarda y vuelve a Reiniciar el servicio de PostgreSQL.
✅ ¡Listo! El candado ha sido cambiado y ya puedes usar tu nueva clave en el instalador de Finaxis.
